### 2.2 已经能跑通的事

#### A. MCP stdio 代理（`sentinel_mcp/proxy.py`）
- newline-delimited JSON-RPC 2.0 拦截 `tools/call`
- 走 Guard：ALLOW 透传 / DENY 回 -32000 error / REDACT 改写 args / **ASK_USER 阻塞等审批**
- 关键修复：Guard.check_tool_call 是同步函数 + 回调内部 sleep-poll，会阻塞 asyncio event loop。
  解法：`await loop.run_in_executor(None, self.guard.check_tool_call, call)`，在线程池里跑同步调用，
  代理就能在等审批的同时继续收发其他 JSON-RPC 消息。

#### B. CLI（`sentinel_mcp/cli.py`）
```
sentinel-mcp wrap [--config policies.yaml] [--audit-db PATH] [--ask-timeout 60] -- <upstream-cmd> ...
```
- 默认策略复用 v0.1 的 `0425_test01/config/policies.yaml`
- 默认审计 DB：`0427_test01/data/sentinel.db`，`SENTINEL_DB` 可覆盖
- `--ask-timeout`（env `SENTINEL_ASK_TIMEOUT`，默认 60s）：审批超时按拒绝处理

#### C. 跨进程审批队列（`sentinel_mcp/approvals.py` — 新增）
- SQLite 表 `pending_decisions`（id / created_at / decided_at / tool_name / args_json / reason / risk_score / triggered / status / decided_by / session_id）
- `journal_mode=WAL`，多进程读写安全
- `PendingDecisions.create()` 写一行，`wait(pid, timeout, poll_interval=0.25)` sleep-poll 直到 status 变化或超时
- `make_callback(timeout_seconds, on_request=None)` 返回闭包，喂给 `Guard.from_yaml(..., ask_user_callback=...)`
- `decide(pid, approved, by="dashboard")` / `list_pending()` / `list_recent()` / `list_since()`：dashboard 用

#### D. PWA Dashboard（`pwa_dashboard/server.py` + `templates/index.html`）
- 端口 `8766`（避开 v0.1 的 8765）
- PWA 三件套：`/manifest.webmanifest` / `/sw.js`（带 `Service-Worker-Allowed: /`）/ 192·512·maskable 图标
- 既有 API：`/api/health` `/api/events` `/api/stats` `/api/clear`
- **新增审批 API**：
  - `GET  /api/approvals?only_pending=true&limit=50`
  - `POST /api/approvals/{pid}/decide`，body `{"approved": bool, "by": "dashboard"|"phone"}`，已处理返回 409
- **新增 SSE**：`GET /api/events/stream`
  - 起手发 `hello` + 最近 50 条历史 + 当前所有 pending
  - 之后每 500ms（`SENTINEL_SSE_INTERVAL` 可覆盖）扫一次 audit + approvals 增量推送
  - 事件名：`hello` / `event`（audit）/ `pending`（新待审批）/ `decided`（审批被处理）
  - 闲时 `: ping\n\n` 心跳防中间代理掐断
  - `request.is_disconnected()` 客户端断线即退出
- **前端**（`templates/index.html`）：
  - SSE 状态指示灯：实时连接中（绿）/ 正在连接（黄）/ 连接断开（红）
  - 「待审批队列」amber 卡片：显示 tool_name / reason / Approve / Deny；点了就 POST decide
  - 浏览器原生 `Notification` 推送（pending 来了弹本地通知）
  - 每 3s 仅刷新统计；事件流由 SSE 推送，不再轮询

### 2.4 期间踩过的坑（避免下次再踩）

1. **本机代理 7890 挂掉**：所有 pip / curl / npm 之前先 `unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY`
2. **starlette 1.0 `TemplateResponse` 签名变了**：必须 `templates.TemplateResponse(request=request, name=...)`，
   旧的 `(name, {"request": ...})` 会抛 `TypeError: unhashable type: 'dict'`
3. **测试 SSE 收不到事件**：`test_real_mcp.py` 之前硬写 `--audit-db /tmp/...`，与 dashboard 的 DB 不是同一个。
   已统一到 `SENTINEL_DB` env，dashboard / proxy / tests 都从这里读
4. **ASK_USER 死锁雏形**：Guard 是同步、callback 内部又要 sleep-poll SQLite，
   直接放 asyncio 协程里会让 event loop 完全停下来。已用 `run_in_executor` 解掉
